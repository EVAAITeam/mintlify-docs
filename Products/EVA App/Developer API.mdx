## Endpoint: Full Token Analysis

Performs a comprehensive security analysis of a smart contract token, including threat detection, security audits, and AI-powered similarity analysis.

```
POST /evmFullanalyze
```

### Request

**Headers:**

```
Content-Type: application/json
x-api-key: YOUR_API_KEY
```

**Body:**

```json
{
  "chain": 56,
  "ca": "0x00e22ed2c31875abaa34ed0659acb76734b9540d"
}
```

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `chain` | number | ‚úÖ Yes | Chain ID (see supported chains below) |
| `ca` | string | ‚úÖ Yes | Contract address (checksummed or lowercase) |

### Response Fields

#### Core Token Information

| Field | Type | Description |
|-------|------|-------------|
| `owner` | string | Current owner address or `"HIDDEN"` if renounced |
| `tokenAddress` | string | Contract address |
| `symbol` | string | Token symbol |
| `name` | string | Token name |
| `chain` | string | Chain ID |
| `decimals` | string | Token decimals |
| `totalSupply` | string | Total token supply (in wei) |
| `contractName` | string | Contract name from source code |
| `isProxy` | boolean | Whether contract is a proxy |
| `age` | number | Token deployment timestamp (milliseconds) |
| `lastchecked` | number | Last analysis timestamp (milliseconds) |

#### `initInfo` ‚Äî Deployment Information

| Field | Type | Description |
|-------|------|-------------|
| `blockNumber` | string | Block number of deployment |
| `contractCreator` | string | Address that deployed the contract |
| `txHash` | string | Deployment transaction hash |
| `timestamp` | number | Deployment timestamp (Unix seconds) |

#### `threats` ‚Äî Detected Threats

Array of strings indicating detected security threats:

| Threat | Description |
|--------|-------------|
| `Honeypot` | Contract designed to trap funds |
| `Shypooning` | Malicious behavior allowing fund manipulation |

```json
"threats": ["Honeypot", "Shypooning"]
```

#### `auditInfo` ‚Äî Security Audit Results

| Field | Type | Description |
|-------|------|-------------|
| `ownershipStatus` | string | Ownership status: `"Active"`, `"Renounced"`, etc. |
| `flags` | array | Array of security check results |

**Flag Object:**

| Field | Type | Description |
|-------|------|-------------|
| `category` | string | Security category (e.g., `"Honeypot"`, `"Tax Issue"`, `"Other Risks"`) |
| `label` | string | Visual indicator: üî¥ = risk detected, üü¢ = passed |
| `description` | string | Detailed description of the finding (empty if passed) |

```json
{
  "category": "Honeypot",
  "label": "üî¥",
  "description": "The contract uses `subSkip` in `transferFrom`, allowing tax wallets to bypass allowance checks..."
}
```

#### `similarityMap` ‚Äî AI-Powered Similar Tokens

Array of up to 5 similar tokens detected through AI analysis.

| Field | Type | Description |
|-------|------|-------------|
| `symbol` | string | Token symbol |
| `id` | string | Contract address |
| `chain` | string | Chain ID where token exists |
| `score` | number | Similarity score (0-1, higher = more similar) |
| `tags` | array | Tags like `"scam"`, `"verified"`, etc. |

```json
{
  "symbol": "AZZA",
  "score": 0.8715265989303589,
  "id": "0xc2d49b3358104c764100a3044aefad672ab794da",
  "chain": "8453",
  "tags": []
}
```

## How It Works

<Steps>
  <Step title="Initial Analysis">
    When you analyze a token, EVA fetches contract data from the blockchain.
  </Step>
  <Step title="Security Audit">
    The contract source code is analyzed for common vulnerabilities and threats.
  </Step>
  <Step title="Similarity Detection">
    AI-powered pattern matching identifies similar contracts across all chains.
  </Step>
  <Step title="Caching">
    Results are cached for efficiency with intelligent invalidation.
  </Step>
</Steps>

<Note>
  **Cache Behavior:**
  - **Renounced tokens:** 1 hour cache (ownership won't change)
  - **Active ownership:** Re-validates owner before serving cache
  - **Similarity data:** Refreshed every 5 minutes
</Note>

</Warning>

async function analyzeToken(chain, contractAddress) {
  const response = await fetch('https://api.eva-ai.cloud/evmFullanalyze', {
    method: 'POST',

```python Python
import requests

def analyze_token(chain, contract_address):
    url = 'https://api.eva-ai.cloud/evmFullanalyze'
    headers = {
        'Content-Type': 'application/json',
        'x-api-key': 'YOUR_API_KEY'
    }
    payload = {
        'chain': chain,
        'ca': contract_address
    }

    response = requests.post(url, headers=headers, json=payload)

    if response.status_code != 200:
        raise Exception(f'API error: {response.status_code}')

    data = response.json()

    # Check for threats
    if data['threats']:
        print(f"‚ö†Ô∏è Threats detected: {data['threats']}")

    # Check red flags
    risks = [f for f in data['auditInfo']['flags'] if f['label'] == 'üî¥']
    if risks:
        print(f"üö® Security risks found: {len(risks)}")
        for risk in risks:
            print(f"  - {risk['category']}: {risk['description']}")

    return data

# Usage
result = analyze_token(56, '0x00e22ed2c31875abaa34ed0659acb76734b9540d')
print(f"Token: {result['name']} ({result['symbol']})")
```

```typescript TypeScript
interface AnalysisRequest {
  chain: number;
  ca: string;
}

interface AuditFlag {
  category: string;
  label: string;
  description: string;
}

interface AnalysisResponse {
  owner: string;
  tokenAddress: string;
  symbol: string;
  name: string;
  chain: string;
  decimals: string;
  totalSupply: string;
  threats: string[];
  auditInfo: {
    ownershipStatus: string;
    flags: AuditFlag[];
  };
  similarityMap: Array<{
    symbol: string;
    id: string;
    chain: string;
    score: number;
    tags: string[];
  }>;
}

async function analyzeToken(
  chain: number,
  contractAddress: string
): Promise<AnalysisResponse> {
  const response = await fetch('https://api.eva-ai.cloud/evmFullanalyze', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': process.env.EVA_API_KEY!
    },
    body: JSON.stringify({ chain, ca: contractAddress })
  });

  if (!response.ok) {
    throw new Error(`API error: ${response.status}`);
  }

  return response.json();
}
```
